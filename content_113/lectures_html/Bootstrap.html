Including Bootstrap

To start using Bootstrap, we will add the following 4 lines in the head of out HTML file.

    <meta name="viewport" content="width=device-width, initial-scale=1">
    <link rel="stylesheet" href="http://maxcdn.bootstrapcdn.com/bootstrap/3.3.6/css/bootstrap.min.css">
    <script src="https://ajax.googleapis.com/ajax/libs/jquery/1.12.0/jquery.min.js"></script>
    <script src="http://maxcdn.bootstrapcdn.com/bootstrap/3.3.6/js/bootstrap.min.js"></script>

The first line sets up the page in a mobile friendly way by setting the page width equal to width of the device. The following 3 lines download the CSS and Javascript files that will run Bootstrap. Bootstrap using jquery functions, so we have to include this as well. With these three files included, we can start using Bootstrap, and jquery, on our page.

Putting this into an HTML file will create the following shell for a Bootstrap powered page. 


<!DOCTYPE html>
<html lang="en">
<head>
    <meta charset="utf-8">
    <meta name="viewport" content="width=device-width, initial-scale=1">
    <link rel="stylesheet" href="http://maxcdn.bootstrapcdn.com/bootstrap/3.3.6/css/bootstrap.min.css">
    <script src="https://ajax.googleapis.com/ajax/libs/jquery/1.12.0/jquery.min.js"></script>
    <script src="http://maxcdn.bootstrapcdn.com/bootstrap/3.3.6/js/bootstrap.min.js"></script>
</head>
<body>
<div class="container">
    <p>Now we're cooking with Bootstrap!</p>
</div>
</body>
</html>


This page is ready to take advantage of Bootstrap, through we haven't done anything interesting with it yet. For the examples on this page, we will not show the includes and the header since this site itself has already included them.

Bootstrap Classes

Most of our interaction with Bootstrap will be through classes. Bootstrap has a multitude of classes defined that will alter the appearance of our HTML elements by changing their class attribute.



We used two Bootstrap classes in this example. The first is a div with the class container. This will be the outer tag for our Bootstrap pages and will handle the top level formatting to keep our elements on the screen regardless of its size. The container class will add padding on the sides of the page. To remove this padding and use the full page width we can use the container-fluid class instead.

The second class uses is text-success which changed the style of our paragraph text.



To add multiple formatting changes to the same element we can add several classes each separated by a space.

The grid

Bootstrap provides a 12 column grid system to help organize our content the way we want. This grid is similar to an HTML table with 12 columns and we will specify how many of the rows columns each cell will use in order to format the page how we want.



It's important that each row adds up to 12 columns or the page might not display properly. We can add empty columns to have blank space on the site and further control the layout.

There are 4 different sizes that can be used for the column. In increasing order of size, they are: {"xs", "sm", "md", "lg"}. Changing the size of a column will change the way it collapses with the smallest size maintaining it's structure even on mobile phones. Try changing these sizes then resizing your browser window to se how the structure changes.
