While AND and OR were both binary boolean operator (they looked at the values of 2 boolean expressions),
        NOT is a unary boolean operator, meaning that it only looks at the value of one boolean expression. NOT
        returns the opposite of the given expression's boolean value.

        alphabetical order:

    

        
        

            


                

                
            

        


        

    


    
Although this code will alphabetize the code, it isn't the best way to solve the problem.
        Right now, the code will run through every possibility until it reaches the one that is true. In the worst
        case, the arrangement that we want could be the very last one that we check for. So to alphabetize n
        strings,
        we could need to look at up to n! (n factorial) cases. For 3 strings that means
        there are at most 6 cases to check. That's not so bad, but what if we had more strings that we wanted to
        alphabetize?
        4 strings would give us 24 cases, 10 strings is 3628800 cases, and 15 strings is 1307674368000 cases!
    

    
We could make this a bit more efficient if we break up the comparisons so that we can skip cases that we know
