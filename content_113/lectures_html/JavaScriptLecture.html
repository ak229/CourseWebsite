JavaScript Syntax

Although JavaScript and Python are both programming languages, and do have many similarities, they are not the same language, and do have some differences that we will need to be aware of while writing code:

Differences from Python

Comments

All comments in Python are preceded by a '#'. However in JavaScript, they are preceded by a '//'.


Variable Declarations

In JavaScript, you must declare all variables before you use them. This is done once for each variable using the var keyword:

var fluffy;
fluffy=9;
fluffy="puppies"

//declaration and assignment can happen in the same line:
var numberOfHappyKittens=100;

//several variables can be declared in the same declaration statement:
var bunnies, moreBunnies, evenMoreBunnies, tooManyBunnies;
bunnies=5;
moreBunnies=17;
evenMoreBunnies=77;
tooManyBunnies=765293;

Terminators

These unfortunately have nothing to do with killer robots from the future. In this case, a terminator is simply a semicolon (;). While in Python, each statement was separated by a new line. In JavaScript, a semicolon is added at the end of each statement to mark its end.

Python:

#Each statement gets its own line
number1=5+6
number2=7

#Writing code like this will give you errors
number1=5+6  number2=7

JavaScript:

//Each statement ends with a semicolon
var number1=5+6;
var number2=7;
// Technically you can write code like this, however it isn't as easy to read
var number1=5+6; var number2=7;

Curly Braces

Curly braces, {}, are used to mark the beginning and end of functions, loops, if statements and else statements. Because of these, JavaScript is not dependant on indentation the way that Python is. However, if you are not careful about your closing braces, you may end up with lines of code inside loops or functions that they were not supposed to be in!

Python:

#This is outside the function
def fancyFunction():
    #This is inside the function
    i=0
    while(i<10):
        #This is inside the while loop inside the function
        i=i+1
    #This is back outside the while loop, but still in the funciton
#This is back outside the function

JavaScript:

//This is outside the function
function fancyFunction(){
//This is inside the function
var i=0;
    while(i<10){
    //This is inside the while loop and inside the function
//This is still inside the while loop and the function
i=i+1;

}
        //This is back outside the while loop, but not outside the function

}
//This is outside the function again

Functions

As seen in the above example, when defining a JavaScript function, the word function is used instead of 'def', as was used in Python.

Boolean Operators

Although the EQUAL operator is the same in JavaScript as it is in Python (==), the OR and AND operators are different. In JavaScript, OR is || and AND is &&.

JavaScript Functions

We'll look at a few fairly simple examples of JavaScript functions before jumping into integrating JavaScript into HTML files

function addTwo(number){
    var answer=number+2;
    return answer;
}

//This function uses the boolean AND operator:

function betweenTwoAndTen(number){
    return ((number>=2)&&(number<=10));
}

//This function uses the boolean OR operator:

function notBetweenTwoAndTen(number){
    return ((number<2)||(number>10))
}

Some functions that we will use when we integrate JavaScript into HTML files:

  • getElementById(elementId)
    • Called on the document object, this function returns a HTML element according to its assigned id value
  • innerHTML
    • Called on a HTML element. Allows the value inside certain tags to be reset (such as paragraph tags)
  • src
    • Called on an image. Allows the source of an image to be reset.
  • value
    • Called on an input box. Returns the value in the input box.

Using JavaScript Functions in a HTML File

JavaScript functions used in an HTML file are all defined within script tags (<script> and </script>)

Buttons

We're going to make a brief return to HTML here to learn how to make buttons. Here is a very boring button:

This button was created with the following line of HTML code:

<button type="button">Uninteresting Button</button>

As is expected from HTML, we start and end our button with the open and close tags for a button. Inside the opening tag, the type of our button, "button", is specfied, and between the opening and closing tags, we see the button label, Uninteresting Button.

Notice that when clicked, this button doesn't do anything. However, with a bit of JavaScript, this button can become much more fun!

The following code was needed to make the more interesting button:

<button type="button" onclick="interestingButtonExample1()">Interesting Button</button>

<p id="ButtonExample"></p>

<script>

function interestingButtonExample1(){
    document.getElementById("ButtonExample").innerHTML ="The button has been pushed!!!!";
}
</script>

Here, the structure of the button is the same, with the exception of adding the onclick element inside the opening button tag. This defines what will happen when the button is clicked. In this case, it is a function call to a function defined in the script section of the HTML file. After the button tags, we see the paragraph tags. Although there is initially nothing in the paragraph tags, we see that the opening tag has an id label set to "ButtonExample".

Inside the script tags, the function called by the onclick section of the button tag is defined. The function, interestingButtonExample1(), accesses the paragraph tags specified above using the id label "ButtonExample", and then sets the inside of the tags to "The button has been pushed!!!!". This changes the HTML so that it displays the text "The button has been pushed!!!!" in the specified spot at the specified time (when the button has been clicked).

AJAX: Asynchronous Javascript

The goal: Make interactive pages that do not require a refresh whenever a user runs a python script on the server.

We have seen how our users can access our python script by directly going to the url that specifies the script and typing the parameters after a question mark in this url. This is great for web API's, but not for users who want to access your site through a browser. To view interactive data through a browser we wil use AJAX to build an HTTP request to our python scripts. 

function getReviews() {
    var xmlhttp = new XMLHttpRequest();
    xmlhttp.onreadystatechange = function () {
        if (xmlhttp.readyState == 4) {
            document.getElementById("reviewResults").innerHTML = xmlhttp.responseText;
        }
    };
    xmlhttp.open("GET", "cgi-bin/interface.cgi?request=getReviews", true);
    xmlhttp.send();
}

In this example we create an HTTP request to "cgi-bin/interface.cgi?request=getReviews" and place the results in an HTML element with id "reviewResults".



We call this function in our HTML with the following code: 

<h3><a href="#" onclick="aggregateReviews(); return false;">
<span class="label label-primary">Get Aggregate Results</span></a></h3>
<div id="aggregateResults"></div>

When the link is clicked, it will call our AJAX function and place the results in the "aggregateResults" without refreshing the page. This makes our site much easier to use since our users do not even have to make their own HTTP requests via custom urls since we do it for them.
