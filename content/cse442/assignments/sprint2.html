#} == Group Points

The following 4 points are evaluated on a group basis. All members of the group will share the same score for these points.

= Group Communication (1 point)

All required meetings have occurred and the minutes for each have been pushed to the group repo. For each sprint, excluding sprint 1, this will include 3 weekly meetings with the groups TA, 3 weekly group meetings without course staff, and 1 meeting with the course instructor. Minutes for all 7 meetings must appear in the group repo. If a single meeting is missed by the entire group, or the minutes are not uploaded, this point will not be awarded.

If the quality of the meetings or minutes are poor this points might not be awarded, but more importantly the minutes will be used to track any issues that the group members have been facing. There are often unforeseen issues and complications while writing code that can turn a 10 minute task into a week-long time sink. If a task is not completed by the end of a sprint but there are well documented complications that appear in the minutes throughout the sprint this can be used to your advantage to potentially get credit for an incomplete task. If a task is not completed and there has been no communication about a complication then no credit can be given for the task.

= Group Version Control (1 point)

The group has been properly applying the branching model used for this course. Specifically, the group has a master and development branch which the group members only merge into and never push to directly. Group members create feature branches off of develop to work on their tasks and merge the branch into develop when the task is complete.

This point can still be awarded even if a few group members are violating the branching model. If the group as a whole is not implementing the branching model this point will not be awarded.

= Group Task Completion / Version Release (1 point)

The next working version of the software has been merged from develop to master and is tagged as release "v0.2.0" in GitHub. The software must have some functionality added since the previous sprint. If at least one group member completed all their tasks and merged them into master then this point is awarded.

= Video (1 point)

Upload a demo of your software to YouTube. This video must give an accurate view of the current state of your project and show the running software. This video will be directed towards a non-technical audience and will show them the current working functionality that has been implemented. You should not be scrolling through code in this video.

This video must be shorter than 2 minutes to get credit for the video. This will be strictly enforced. The goal is to quickly communicate the status of your product to the end user.
