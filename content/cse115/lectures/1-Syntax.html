= print()

  • Print text to the screen (console)

  • print()

The print function will print its arguments (whatever is contained between the parentheses) to the console. For example, when run:

[.python_code] -- print(5) print("puppies") --
Note that each call of print adds a new line to the output.
== Comments

  • Ignored by the interpreter

  • Write notes for humans to read

Sometimes it is helpful to add notes to your code. However, it is important to differentiate between lines of comments and lines of code. Otherwise, the compiler may run into trouble when it tries to execute comments as code.

In Python, adding a # character before a line marks it as a comment. The interpreter will simply skip over this line when it executes the code.


[.python_example] -- #This is an example print("This line will be printed") #print("This line will not be printed") --

Comments can also be added after a line of code.

[.python_example] -- #This is an example print("Python is a great programming language") # Java is pretty cool too -- == Variables

  • Store a value

  • Use it later

Variables are used to refer to values. In Python, the type of a variable does not need to be declared before it can be used, and so can hold objects of differing types.

A variable is assigned a value using the assignment operator, = , in the following manner:

variableName=5

This line of code creates a variable named "variableName" and assigns it the value of 5. The variable name can be anything that the programmer chooses, but should be something that describes what the variable is storing. Using a variable allows the programmer to use a value without accessing the value explicitly.
